# Lower_Bounding_Spectral_Gaps

This repository contains the code and data for the computational study corresponding to Section 7 of the paper: *"A Matrix Analysis Approach to Spectral Gap Estimation in Adiabatic Quantum Computing"*.

The code implements various rigorous theoretical lower-bound certificates (Weyl-Lipschitz, PSD Floor, Horn $T^n_1$) for the spectral gap of Adiabatic Quantum Computing (AQC) paths, specifically targeting Max-Cut on random graphs.

## Repository Structure & File Overview

### Core Libraries & Benchmarking
- **`maxcut_gap_benchmark.py`**: The primary computational engine. It implements the theoretical certificates (Weyl-Lipschitz [W-Lip], PSD Floor [PSD], Horn [Horn]), exact diagonalization methods for the ground truth, and other baseline estimates (Krylov, 2-level model). It handles the generation of Max-Cut paths and outputs benchmark results.
- **`main.py`**: Contains fundamental helper functions for instance generation (Erdős-Rényi graphs) and cost vector definitions (e.g., `pinned_cost_vector`), as well as classical local search heuristics for the Max-Cut problem.

### Experiments & Data Generation
- **`generate_updated_comparison.py`**: A dedicated script to generate a specific detailed comparison of the Weyl and Horn bounds on a single random instance (N=10). It evaluates bounds with and without adaptive anchors, generating visual plots to compare certified areas.
- **`horn_anchors_test.py`**: An experiment script that specifically tests the Horn $T^n_1$ bound's performance as the number of anchors (grid points where the exact spectrum is computed) increases, demonstrating how the bounds tighten with more spectral information.

### Visualization & Plotting
- **`plot_new_results.py`**: Reads the aggregated experimental data (from `section7_results.csv`) and generates summary plots. This includes the oracle query efficiency comparison (Hybrid Certified Sweep vs. Uniform Grid) and the fractional path coverage of different certificates across varying system sizes (Qubits N).

### Data & Outputs
- **`section7_results.csv`**: Contains the raw numerical benchmarking metrics generated by the scripts across different instances, methods, and system sizes.
- **`results/`**: (Directory) Output destination for the generated plots and figures such as `efficiency_comparison.png`, `horn_anchors_test.png`, and `comparison_updated_N10_no_anchors.png`.

## Workflow: How the scripts work together

1. **Instance Generation**: Scripts like `maxcut_gap_benchmark.py` or `generate_updated_comparison.py` use methods (often from `main.py`) to instantiate random Max-Cut problems.
2. **Path Construction**: They build the Adiabatic Quantum Computing Hamiltonian path $H(s) = (1-s)H_I + sH_P$.
3. **Benchmarking/Certificates**: The algorithms defined in `maxcut_gap_benchmark.py` and `horn_anchors_test.py` compute the spectral gaps exactly and evaluate the various lower bounds (Weyl, Horn, PSD) across the annealing parameter $s \in [0, 1]$.
4. **Data Aggregation**: System-level results and timings are aggregated into `section7_results.csv`.
5. **Visualization**: Visualization scripts (`plot_new_results.py`, `generate_updated_comparison.py`) read the raw data or run on-the-fly experiments to generate `.png` plots in the `results/` directory, directly producing the figures presented in the research.

## How to Check the Data and Reproduce Plots

- **To view the aggregated benchmarking results**, inspect the `section7_results.csv` file.

Here is exactly which script you need to run for each graph:

- **`efficiency_comparison.png`** (and other summary plots): Generated by running:
  ```bash
  python plot_new_results.py
  ```
- **`comparison_updated_N10_anchors.png`**: Generated by running:
  ```bash
  python generate_updated_comparison.py
  ```
- **`grid_density_vs_gap.png`**: Generated by running:
  ```bash
  python scratch/plot_grid_vs_gap.py
  ```
  *(Note: this saves the files as `grid_density_vs_gap_N12.png` and `grid_density_vs_gap_N14.png`).*
- **`horn_anchors_test.png`**: Generated by running:
  ```bash
  python horn_anchors_test.py
  ```

If you want all of these graphs, you will need to run those separate Python commands.

- **To run the full benchmark suite** and generate new data for different instances, you can execute `maxcut_gap_benchmark.py`. This script provides a command-line interface; run `python maxcut_gap_benchmark.py -h` for available arguments (like `--scan-N` or `--full-space`).
